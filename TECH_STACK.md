# 项目技术栈介绍

## 1. 项目定位

本项目是一个面向中文小说问答场景的 Agentic RAG 系统，结合了大模型推理、混合检索、重排序与可视化调试能力，用于提升长文本知识问答的准确率与可解释性。

---

## 2. 核心技术栈

### 2.1 编排与智能体框架

- **LangGraph**
  - 用于构建多步推理与检索工作流
  - 支持 planner、executor、reflector、generator、verifier 等节点编排
  - 便于实现多轮检索、反思与答案生成闭环

- **LangChain 生态**
  - 用于对接大模型、提示词和工具链
  - 适合作为 Agent 与检索模块之间的连接层

### 2.2 检索系统

- **Qdrant**
  - 负责向量检索
  - 存储 chunk embedding，并支持相似度召回
  - 适合处理语义相近但表述不同的中文文本

- **Elasticsearch**
  - 负责 BM25 关键词检索
  - 适合召回专有名词、人物名、地名、事件名等精确匹配内容

- **混合检索（Hybrid Retrieval）**
  - 将向量检索与关键词检索结果融合
  - 通过 RRF（Reciprocal Rank Fusion）进行排序合并
  - 提升召回率与鲁棒性

### 2.3 重排序

- **Reranker 模型**
  - 对混合检索后的候选结果进行二次排序
  - 用于筛掉语义相关性较弱的 chunk
  - 提升最终上下文质量，降低噪声输入给大模型的概率

### 2.4 文本预处理与切分

- **中文分词 / 预分词**
  - BM25 索引阶段使用 `jieba` 进行中文预分词
  - 缓解中文无空格分词带来的检索问题

- **Chunk 切分策略**
  - 将长文本切成适合向量检索的块
  - 支持最大长度、重叠窗口、最小长度等控制参数
  - 便于兼顾召回精度与上下文完整性

### 2.5 向量表示

- **SentenceTransformers**
  - 用于加载中文向量模型并生成 embedding
  - 适合本地批量编码与索引构建

- **中文嵌入模型**
  - 默认使用 `BAAI/bge-large-zh`
  - 适合中文语义检索任务

- **中文重排序模型**
  - 默认使用 `BAAI/bge-reranker-large`
  - 用于对候选 chunk 进行相关性打分

### 2.6 后端数据存储

- **JSON 持久化文件**
  - `index/novel_chunks.json` 作为 chunk 的持久化中间层
  - 便于增量更新、重建索引与调试

### 2.7 前端展示

- **Streamlit**
  - 用于构建交互式 Web 调试界面
  - 支持上传文件、触发索引、查看检索轨迹和节点耗时
  - 适合快速验证 RAG 流程效果

### 2.8 语言与运行环境

- **Python 3.10+**
  - 作为项目主语言
  - 适合快速开发 NLP、RAG 与 Agent 流程

## 3. 外部服务依赖

### 3.1 Qdrant

- 端口：`6333`
- 用途：向量数据库
- 场景：存储 chunk 向量并进行相似度召回

### 3.2 Elasticsearch

- 端口：`9200`
- 用途：关键词检索引擎
- 场景：BM25 精确召回与补充召回

### 3.3 大模型 API

- 默认接入 **DeepSeek API**
- 用途：
  - Planner 生成检索计划
  - Generator 生成答案
  - Verifier 做结果校验或反思

---

## 4. 项目技术架构

整体流程可以概括为：

1. 上传文本或加载语料
2. 文本预处理与切分
3. 生成向量并写入 Qdrant
4. 生成 BM25 文本索引并写入 Elasticsearch
5. 用户提问后进入 Agent 工作流
6. Planner 负责规划检索路径
7. Executor 调用混合检索与 rerank
8. Generator 基于上下文生成答案
9. Verifier 对答案进行检查与修正
10. 前端展示最终回答和检索 trace

---

## 5. 技术特点

- **中文场景友好**：针对中文小说和中文知识文本优化
- **混合检索增强**：兼顾语义召回与关键词召回
- **可解释性强**：支持检索轨迹与节点耗时展示
- **支持增量更新**：可追加新文档，也可覆盖重建索引
- **适合实验迭代**：配置集中，便于快速调参和优化

---

## 6. 适用场景

- 中文小说问答
- 长文本知识检索
- 多轮对话式 RAG
- 检索策略与分块策略实验
- Agent 工作流教学与演示

---

## 7. 一句话总结

这是一个以 **LangGraph Agent + Qdrant 向量检索 + Elasticsearch BM25 + Reranker** 为核心的中文 RAG 项目，重点解决中文长文本场景下的召回、排序与答案生成问题。
